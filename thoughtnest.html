üîç Key Decisions

Why Hibernate?

Hibernate enabled me to avoid boilerplate SQL while managing complex DB relationships through annotations like @Entity, @OneToMany, and transaction boundaries in service methods. It streamlined data access and kept repository code concise (findByTitle(), deleteById()).

Why H2 for Testing?

H2‚Äôs in-memory design made it ideal for fast, isolated integration tests. Combined with Hibernate, it automatically validated my DB schema and controller behavior ‚Äî without touching production data.

üîê Security Highlights

  • All secrets & credentials stored via environment variables
  • DTO usage to prevent direct exposure of entity models
